# Spec Driven Checklist

Use this checklist when operating with `memory-bank/spec.md`, `memory-bank/plan.md`, and `memory-bank/task.md`.

## Spec

- Is the goal explicit?
- Are success criteria and non-negotiables written down?
- Are long-lived constraints separated from temporary execution notes?

## Plan

- Is there one active slice?
- Is the validation method explicit?
- Is the next slice listed but not merged into the active slice?

## Task

- Does each task have `Status`, `Owner`, `Next step`, and `Handoff note`?
- Is the task small enough for one reviewable change?
- Are debt, evidence, and open questions written where the next agent can find them?

## Discipline

- Update docs when behavior or workflow changes.
- Split tasks before implementation if they span multiple unrelated behaviors.
- Keep `memory-bank/` as the shared handoff source of truth.
